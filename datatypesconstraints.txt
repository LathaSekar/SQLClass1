1. Number Data Type

NUMBER is used to store numeric values (integers, decimals, positive, negative).

Syntax
NUMBER(p,s)


*p → precision (total digits)
*s → scale (digits after decimal)

Example

CREATE TABLE employee (
   emp_id NUMBER(5),
   salary NUMBER(8,2)
);

Example values

emp_id = 101
salary = 25000.50
 2. Character Data Types

Used to store text or strings.

### Types

| Data Type | Description               |
| --------- | ------------------------- |
| CHAR      | Fixed length character    |
| VARCHAR2  | Variable length character |
| NCHAR     | Unicode fixed length      |
| NVARCHAR2 | Unicode variable length   |

 Example

CREATE TABLE student (
   name VARCHAR2(50),
   gender CHAR(1),
   firstname NCHAR(5),   max size 2000
   lastname NVARCHAR2(10) max size 4000
);

Example values

name = 'Ravi'
gender = 'M'
firstname='Ajay'
lastname='Ajay'

LONG and ROW Data Types
LONG
Used to store large text data.
* Maximum size → 2GB
* Only one LONG column per table

Example
CREATE TABLE documents (
   description LONG
);

ROWID
ROWID is a unique address of a row in a table.
It tells Oracle where the row is stored physically.

Example

SELECT ROWID, name 
FROM employees;
Output example

AAAPzEAAEAAAACXAAA   Ravi

 4. Date and Time Data Types

Used to store date and time values.

Types

| Data Type                | Description                         |
| ------------------------ | ----------------------------------- |
| DATE                     | Stores date and time                |
| TIMESTAMP                | Stores date with fractional seconds |
| TIMESTAMP WITH TIME ZONE | Includes timezone                   |
| INTERVAL                 | Stores time duration                |

Example

CREATE TABLE orders (
   order_date DATE
);

Insert example

INSERT INTO orders VALUES (SYSDATE);

Example 
output

5. Large Object Data Types (LOB Types)

 Types

 6. Pseudo Columns
Pseudo columns behave like columns but are not actually stored in the table.

They give special information about rows.

 Common Pseudo Columns

| Pseudo Column | Description                |
| ------------- | -------------------------- |
| ROWID         | Address of row             |
| ROWNUM        | Row number in query result |
| SYSDATE       | Current system date        |
| LEVEL         | Hierarchical query level   |

Example 1 – ROWNUM
SELECT * 
FROM employees
WHERE ROWNUM <= 5;
Shows first 5 rows.
Example 2 – SYSDATE
SELECT SYSDATE FROM dual;
Output
10-MAR-26
Example 3 – ROWID
SELECT ROWID, name 
FROM employees;

SQL Constraints
Constraints are used to restrict the type of data that can be inserted into a table.
They help to maintain valid and reliable data.
Types of SQL Constraints
Primary Key
Foreign Key
Unique
Not Null
Check
Default

Primary Key Constraint
A Primary Key uniquely identifies each record in a table.
Cannot contain NULL values
Must be unique
Only one primary key per table
Example:CREATE TABLE employees (
    emp_id NUMBER PRIMARY KEY,
    name VARCHAR2(50),
    salary NUMBER
);
Composite Primary Key:
A Composite Primary Key is a primary key formed by combining two or more columns to uniquely identify a record.
CREATE TABLE order_details (
    order_id NUMBER,
    product_id NUMBER,
    PRIMARY KEY(order_id, product_id)
);
Foreign Key Constraint:
A Foreign Key creates a relationship between two tables.
It refers to the Primary Key of another table.
CREATE TABLE departments (
    dept_id NUMBER PRIMARY KEY,
    dept_name VARCHAR2(50)
);
CREATE TABLE employees (
    emp_id NUMBER PRIMARY KEY,
    name VARCHAR2(50),
    dept_id NUMBER,
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id)
);
Rule       What Happens
CASCADE    Child rows are deleted/updated automatically
           If the parent row is deleted or updated, the child rows are automatically deleted or updated.
SET NULL   Child FK column is set to NULL
           If the parent record is deleted, the foreign key in child table becomes NULL.
RESTRICT   Parent cannot be deleted if child exists
           Parent record cannot be deleted if child records exist.
NO ACTION  Same as RESTRICTSET 
           prevents deletion of parent row if child rows exist.
SET DEFAULT    Child FK set to its default value
          If parent row is deleted, the child foreign key column is set to its default value.
